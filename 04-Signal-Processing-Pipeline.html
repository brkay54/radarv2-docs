Signal Processing Pipeline¶

Scope

The AERIS-10 10-stage FPGA receive pipeline, the math each stage performs, and

how the stages fold into radarv2's four RX/DET nodes. This is the heart of the

example — the rest of the system exists to feed and drain this chain.

1. The AERIS-10 FPGA pipeline (reference)¶

The real receive pipeline runs in radar_receiver_final.v across three clock

domains (400 MHz ADC, 100 MHz system, 100 MHz FT601). Ten stages convert raw ADC

samples into a thresholded range-Doppler map.

StageFunctionIn → Out
1ADC acquisition (LVDS→CMOS, clock recovery)8-bit LVDS → 8-bit CMOS
2CDC ADC→processing8-bit synchronized
3Digital down-conversion (NCO mix + CIC + FIR)8-bit → 18-bit I/Q baseband
4DDC output interface (scale/align)18-bit → 16-bit I/Q
5Reference chirp load + latency buffersegment select → delayed 16-bit I/Q ref
6Matched filter (overlap-save pulse compression)16-bit I/Q → 1024-bin range profile
7Range-bin decimation (peak-detect 1024→64)16-bit I/Q → 64 bins
8Doppler processing (Hamming + 32-pt FFT over M chirps)64 bins × 32 chirps → 64×32 map
9Threshold detection (placeholder, not true CFAR)16-bit I+Q → 1-bit flag
10USB output (FT601 packets)range + Doppler + detection → 32-bit packets

Stage math worth knowing¶

  • DDC (stage 3) — an NCO quadrature mixer translates the 120 MHz IF to baseband.

The 32-bit NCO phase increment is $\Delta\phi_\text{NCO} = (f_\text{IF}/f_s)\cdot 2^{32}

= 0.3 \cdot 2^{32} = $ 0x4CCCCCCD. A 5-stage CIC decimates by 4 (DC gain

$4^5 = 1024 = 2^{10}$, compensated by a 10-bit right shift), then a FIR corrects

the CIC's $\text{sinc}^5$ passband droop.

  • Matched filter (stage 6) — frequency-domain pulse compression by overlap-save:

forward FFT of the buffered chirp → element-wise multiply with the reference

chirp's FFT → inverse FFT → time-domain range profile. The long chirp's 3000

samples are processed as 4 segments of 1024 (overlap 128, advance 896); the short

chirp's 50 samples are zero-padded to one 1024-segment. A 3187-cycle latency

buffer aligns the reference with the FFT pipeline.

  • Range decimation (stage 7) — keep the peak-magnitude bin out of each group of

16, reducing 1024 → 64 range bins while preserving target peaks.

  • Doppler (stage 8) — accumulate M chirps per range bin in chirp-major memory

(addr = chirp_index·64 + range_bin), apply a 32-point Hamming window (Q15), run a

32-point FFT per range bin → the range-Doppler map. Output is packed {Q, I} with

range_bin 0–63, doppler_bin 0–31.

  • Detection (stage 9) — the shipped detector is a placeholder: $|I|+|Q| > 10000$

(L1 magnitude vs. a fixed threshold), with misleading cfar_* signal names. There

are no reference cells and the false-alarm rate is uncontrolled. radarv2 replaces

this with a real CA-CFAR (see 07-Detection-and-Tracking).

2. radarv2's RX/DET chain¶

radarv2 folds the ten stages into four nodes. The fast-time path (per chirp) flows

through RxSynthesizer → PulseCompressor; the slow-time path (per dwell, M chirps)

is completed by DopplerProcessor, then CfarDetector thresholds the map.

Two timescales, one diagram¶

The single most important thing to "get" about this pipeline is that it runs on **two

clocks**: a fast-time loop that fires once per chirp (range), and a slow-time

boundary that completes once per dwell of M chirps (velocity). This sequence shows

both:

sequenceDiagram autonumber participant SC as Scan participant RX as RxSynth participant PC as PulseComp participant DP as Doppler participant CF as Cfar rect rgb(225,235,255) loop per chirp (fast time) x32 SC->>RX: ScanCommand (chirp k) RX->>PC: IqFrame PC->>DP: RangeProfile (64 bins) end end rect rgb(255,244,214) DP->>CF: RangeDopplerMap (64x32, at frame_start) CF->>CF: CA-CFAR threshold end
Reading the sequence

The blue band is the fast-time loop: 32 chirps each flow Scan → Rx → PulseComp →

Doppler, building one range profile per chirp. The amber band is the slow-time

boundary: only after all 32 chirps land does Doppler emit the full map and Cfar

threshold it. frame_start on the dwell's first ScanCommand is what resets the

Doppler accumulator. (No box grouping per the vault mermaid rules — rect bands

instead.)

What each node computes¶

RxSynthesizer (stages 1–4 — the radar "world model").

For each ScanCommand it walks the active scenario, selects targets inside the beam

(within the HPBW around the commanded azimuth/elevation), and for each builds the

baseband contribution of one chirp:

  • range → round-trip delay $\tau = 2R/c$ → a beat phase ramp across the 256

fast-time samples (chirp rate $\mu = B/T_c$);

  • radial velocity → a Doppler phase advanced per chirp index (slow-time);
  • two-way SNR via the radar equation, anchored to a reference-range SNR:

$\text{SNR} = \text{SNR}_\text{ref} + \sigma_\text{dBsm} - 40\log_{10}(R/R_\text{ref})$;

  • additive complex Gaussian noise at the configured floor;
  • optional micro-Doppler / JEM (RS3, micro_doppler): rotating jet-engine parts

add a comb of symmetric Doppler sidebands around the body line, keyed off the

target type (a fighter carries a stronger, faster blade-pass comb than an airliner);

  • optional sea-surface multipath (RS3, sea_multipath): at low grazing angles the

direct and sea-reflected rays interfere, so the two-way return is modulated by the

lobing factor $F^2$ ($F=\lvert 1+\rho e^{-j\Delta\phi}\rvert$, $\Delta\phi=4\pi h_r h_t/\lambda R$)

— elevation fading for a sea-skimmer;

  • optional terrain-shaped ground clutter (RS1, ground_clutter): one

near-zero-Doppler (DC) clutter echo per illuminated range bin, sampled from the

MapLayer DTED grid so ground returns follow the

real terrain rather than a flat floor. A constant-$\gamma$ backscatter

$\sigma^0=\gamma\sin(\text{grazing})$ is shaped by the local grazing angle

(depression $\pm$ radial terrain slope — a face tilted toward the radar returns

more), gated by the 4/3-Earth horizon and the beam's elevation extent, and

LOS-occluded by nearer ridges. Being stationary it integrates at the DC Doppler

bin, so it competes with zero-Doppler targets and motivates the Doppler/CFAR

separation downstream.

Output: IqFrame — 256 complex int16 samples plus the beam/dwell/chirp ids.

PulseCompressor (stages 5–7 — matched filter + decimation).

Correlates the IqFrame with the reference chirp (time-domain matched filter at this

size), producing a range profile, then peak-decimates to 64 complex range bins.

The reference bank is synthesized in compress() by default, but can instead be loaded

at on_configure from a binary asset (param matched_filter_table, generated by

scripts/gen_matched_filter_table.py) — the host/build-time analog of the upstream

FPGA chirp-reference block-RAM, loaded the same way MapProvider loads a DTED tile.

Output: RangeProfile — 64 complex int16 bins + range_res (and the mono_az

monopulse voltage carried through from the IqFrame).

DopplerProcessor (stage 8 — coherent integration).

Accumulates the M = 32 same-waveform chirps of a dwell (reset on frame_start),

applies a selectable slow-time taper (RS2: window: rect|hann|hamming|blackman,

default Hamming — rect gives the narrowest mainlobe with −13 dB Doppler sidelobes,

Blackman drives the sidelobes far lower at a wider mainlobe), runs a 32-point DFT per

range bin, and fftshifts the Doppler axis so bin 16 = zero-Doppler (DC) — this is what

makes the centered velocity formula $(\text{doppler\_bin} - 16)\cdot v_\text{res}$ in

07-Detection-and-Tracking valid (bins 0–31 → −16…+15). Output, at the dwell

boundary: RangeDopplerMap — a 64 (range) × 32 (Doppler) complex int16 map +

range_res ($=R_\text{max}/64$ bin spacing) and velocity_res. (radarv2 integrates 32

coherent same-mode chirps rather than the upstream's mixed 16+16 burst — see

I-1

and the dwell decision.)

CfarDetector (stage 9 — CA-CFAR).

Slides a CA-CFAR window over the range-Doppler magnitude map (configurable

$N_\text{ref}$, $N_\text{guard}$, $P_\text{fa}$), declares detections where the cell

under test exceeds the adaptive threshold, and converts each detected cell to a

physical detection: range from the range bin, radial velocity from the Doppler bin,

azimuth/elevation from the beam geometry, SNR from the cell-to-noise ratio. It also

applies terrain masking / line-of-sight from the

MapLayer — a detection whose geometry is occluded by

terrain (below the radar horizon behind a ridge) is dropped. Output:

DetectionReport — up to 16 RadarDetection elements.

3. Fidelity notes¶

Where radarv2 stays faithful and where it simplifies
  • Faithful: the stage order and data products (Iq → range profile → RD map →

detections), the matched-filter and Doppler-FFT transforms, the 64×32

dimensions, the Hamming window, and the M = 32 CPI.

  • Simplified: fixed-point bit widths (int16/float32 software math), the

1024-point FFT + overlap-save segmentation (radarv2 matched-filters at the

smaller demo size directly), the three clock domains and CDC, and the NCO/CIC/FIR

internals of the DDC (modelled as ideal down-conversion).

  • Upgraded: the placeholder fixed-threshold detector → real CA-CFAR.

The dimensions and the transform identities are what make the message catalog and

the algorithms line up with the upstream pipeline doc stage-for-stage. Continue to

05-Node-Architecture.
