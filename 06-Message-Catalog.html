Message Catalog¶

Scope

Every radarv2 aros message type — fields, units, sizes, diagtree paths — and the

upstream AERIS-10 wire structure each one models. All types live in the

radarv2_msgs package, domain aeris, and are codegen'd into the single

aros::schemas registry.

Format reminder

An aros .msg is YAML frontmatter (`kind/name/producer/plane/domain/pool_size/

diagtree_path) + an IDL body of type name[count] @units=… # comment`. Nested

std types are embedded by value (std_msgs/Header) and listed by bare name in

the package glossary.md. Fixed-capacity arrays pair Type field[N] with a

uint32 count. Scalar primitives: int8/16/32/64, uint8/16/32/64,

float32/64, bool.

2. Type definitions¶

ScanCommand¶

Models the STM32 scan FSM output + beam steering + the waveform fields of the host
RadarSettings. One per chirp; frame_start marks a dwell (CPI) boundary.
diagtree: /AERIS10/SCAN/scan_controller/scan_command
std_msgs/Header  header
uint32  scan_id                 # revolution counter
uint32  dwell_id                # monotonic beam-dwell counter
uint32  chirp_index             # 0..M-1 within the dwell
uint16  az_position             # 0..49 azimuth step
uint16  el_position             # 0..30 elevation beam position
float32 beam_azimuth   @units=rad
float32 beam_elevation @units=rad
uint8   waveform_type           # 0 long chirp, 1 short chirp
float32 bandwidth      @units=Hz   # chirp sweep B -> range resolution
float32 pulse_width    @units=s    # chirp duration T_c
float32 prf            @units=Hz
uint32  chirps_per_dwell        # M (CPI length)
uint8   frame_start             # 1 on the first chirp of a dwell (Doppler reset)

IqFrame¶

Models the DDC baseband I/Q for one chirp (stages 1–4). 256 complex samples is a
tractable stand-in for the decimated fast-time record.
diagtree: /AERIS10/RX/rx_synthesizer/iq_frame

uint32  dwell_id

MapLayer¶

Models the loaded physical map (FD-2): a DTED terrain elevation grid plus a bounded
set of road/coastline vector features. Re-published periodically by MapProvider
(once per scan revolution) — aros has no latched/transient-local transport, so a
static map cannot be "published once" and still reach late or cross-host subscribers;
this is the backlog item LATCH-01. Consumed by RxSynthesizer (clutter),
CfarDetector (terrain masking / LOS), and HostEgress (visualization overlay). The
grid is row-major index = row * grid_cols + col.
diagtree: /AERIS10/MAP/map_provider/map_layer
std_msgs/Header  header
uint16  grid_rows                # = 64
uint16  grid_cols                # = 64
float64 origin_lat   @units=deg  # SW corner of the tile
float64 origin_lon   @units=deg
float32 cell_spacing @units=m    # grid posting
float32 min_alt      @units=m
float32 max_alt      @units=m
int16   height[4096]             # 64*64 elevation samples (m, row-major)
uint32  feature_count            # valid leading entries (<=64)
MapFeature features[64]
The asset-loading dependency

MapProvider reads its DTED tile and vector maps from files at on_configure.

aros has no portable node asset-loading mechanism today — this is the backlog

item ASSET-01 (aros/BACKLOG.md). Until it lands, the loader is an interim

Linux-only std::ifstream reader, isolated and clearly marked non-portable, the

same way replay quarantines its file I/O.

3. Mapping to AERIS-10 wire structures¶

radarv2's messages are the aros-typed equivalents of the upstream's hand-packed

binary frames. The table records the correspondence so the model is auditable

against the source.

Host→STM32 RadarSettingscommandScanCommand waveform fields (bandwidth, pulse_width, prf, chirps_per_dwell) + paramsSettings that are static per run live in params; per-dwell waveform travels in the command
STM32→host GPS binary packet ("GPSB", 30 B, big-endian)telemetryPlatformNav (lat/lon/alt/pitch + attitude + ENU)Checksum/endianness are transport concerns; the payload is the model

FT601 radar packet — Doppler I/Q (4×32-bit, {real,imag})dataRangeDopplerMap cells (int16 i/q)Aggregated into the per-dwell map

What does not travel on the wire

As in core aros, the DiagTree path string is never serialized; routing uses the

fnv1a_32 of the node path. The 24-byte MsgHeader (msg_type_id, reserved

pool_index/generation, t_capture, seq, source_id) is prepended by the

framework, not declared in the .msg body.

4. Size and pool budget¶

All container types stay well under the 64 KB message cap (which is a hard limit — the

framework's payload_size is uint16_t, so any POD > 65535 B would silently

truncate; codegen should hard-error above it). MapLayer (~10 KB: int16 height[4096]

= 8192 B + MapFeature features[64]) is the largest, then RangeDopplerMap (~8 KB:

int16 i[2048] + int16 q[2048] = 8192 B + header/scalars). Pool sizes (pool_size

in frontmatter) are set per the in-flight depth of each stage — small for the per-dwell

containers, a few deep for the per-chirp IqFrame/RangeProfile so the RX chain can

have several chirps in flight.
