Upstream Reference — AERIS-10 / PLFM_RADAR¶

This folder is a vendored, read-only mirror of the theory documentation from the

upstream radar project that radarv2 models. The radarv2 example is a software

re-implementation (on the aros executor model) of AERIS-10's receive signal-processing

chain; these documents are the primary source the per-algorithm READMEs cite.

Do not edit these files. They are an upstream copy kept here so radarv2's
educational READMEs can link to stable, in-repo theory instead of an external URL that
may move or go private. To refresh, re-copy from the upstream repo (see below).

What each document covers¶

00_notation/ — symbol and parameter conventions¶

  • symbol_table.md — every symbol used in the theory docs (carrier, chirp, range, Doppler, antenna, detection).
  • parameter_table.md — numerical values for both AERIS-10 variants (Nexus 3 km, Extended 20 km).
  • conventions.md — equation-formatting and sign conventions.

01_physics/ — first-principles derivations¶

  • 01_fmcw_theory.md — wave propagation, radar range equation, beat frequency with Doppler coupling, range/velocity measurement. (→ rx_synthesizer, doppler_processor)
  • 02_lfm_waveform_model.md — LFM chirp math, matched filter, compressed-pulse sidelobe structure. (→ pulse_compressor, window)
  • 03_beamforming_theory.md — ULA array factor, electronic steering, grating lobes, beamwidth. (→ scan_controller, geometry)
  • 04_detection_theory.md — SNR, Swerling models, CFAR threshold derivation, P_d/P_fa curves. (→ cfar_detector, plot_conditioner)
  • 05_noise_analysis.md — thermal noise, noise figure, integration gain. (→ rx_synthesizer)
  • 06_calibration_theory.md — amplitude/phase calibration of the array.

02_hardware/ — system context (selected docs are algorithmically relevant)¶

  • 04_antenna_beamforming.md — array geometry / phase-shifter steering. (→ scan_controller)
  • 07_timing_budget.md — dwell / PRI / revolution timing. (→ scan_controller, doppler_processor)
  • 09_gps_imu_transforms.md — GPS/IMU → world-frame coordinate transforms. (→ nav_source, host_egress)
  • (remaining hardware docs are mirrored for completeness but radarv2 does not model the RF/power hardware.)

03_software/ — the on-board reference implementation¶

  • 01_fpga_pipeline.md — the 10-stage AERIS-10 FPGA chain (ADC → DDC → matched filter → Doppler → MTI → CFAR). radarv2's signal-processing nodes mirror these stages. (→ pulse_compressor, doppler_processor, cfar_detector)
  • 02_stm32_firmware.md, 03_python_gui.md, 04_usb_protocol.md — host/MCU context.

04_research/ — improvement surveys (radarv2's "future realism" hooks map here)¶

  • 01_cfar_variants.md — CA/GO/SO/OS-CFAR comparison. (→ cfar_detector RS2)
  • 02_clutter_rejection.md — MTI/clutter maps. (→ rx_synthesizer clutter, RS1)
  • 03_range_extension.md — integration / range budget.
  • 06_pulse_compression.md — NLFM, windowed matched filter, stretch processing. (→ pulse_compressor, window)
  • 07_target_tracking.md — association, Kalman/IMM tracking. (→ tracking, fusion)
  • 08_adaptive_beamforming.md — adaptive nulling / DBF. (→ scan_controller, future)
  • 04_fpga_optimization.md, 05_ml_detection.md — implementation / ML detection surveys.

How radarv2 cites these¶

Each examples/radarv2/packages/<pkg>/README.md ends with a References section that

links the algorithm it implements to the relevant document(s) here (relative path from the

package up into the vault), plus the upstream GitHub URL for the canonical copy.
